UC1 – S'INSCRIRE EN TANT QU'UTILISATEUR
================================================================

Élément              Description
─────────────────────────────────────────────────────────────
Cas d'utilisation    S'inscrire en tant qu'utilisateur
Acteur principal     Utilisateur non connecté
Pré-conditions       L'utilisateur n'a pas encore de compte
Post-conditions      Le compte est créé et l'utilisateur est connecté

Enchaînement nominal :
1. L'utilisateur accède à la page d'authentification.
2. Il sélectionne l'onglet "Inscription".
3. Il saisit son nom complet, son adresse e-mail et son mot de passe.
4. Il valide le formulaire.
5. Le système confirme la création du compte.
6. L'utilisateur est automatiquement connecté et redirigé vers la page d'accueil.

Enchaînements d'exception :
E1 : L'adresse e-mail est déjà associée à un compte existant
     → Le système affiche un message d'erreur et invite l'utilisateur
       à utiliser une autre adresse ou à se connecter.

UC2 – S'AUTHENTIFIER EN TANT QUE CLIENT
================================================================

Élément              Description
─────────────────────────────────────────────────────────────
Cas d'utilisation    S'authentifier en tant que client
Acteur principal     Utilisateur inscrit
Pré-conditions       L'utilisateur possède un compte actif
Post-conditions      L'utilisateur est connecté et accède à son espace personnel

Enchaînement nominal :
1. L'utilisateur accède à la page de connexion.
2. Il saisit son adresse e-mail et son mot de passe.
3. Il clique sur "Se connecter".
4. Le système vérifie les identifiants.
5. L'utilisateur est redirigé vers la page d'accueil.

E2 : Adresse e-mail inconnue
     → Le système affiche "Aucun compte trouvé avec cet email".

UC3 – RECHERCHER UN RESTAURANT
================================================================

Élément              Description
─────────────────────────────────────────────────────────────
Cas d'utilisation    Rechercher un restaurant
Acteur principal     Utilisateur connecté
Pré-conditions       L'utilisateur est connecté
Post-conditions      Une liste de restaurants correspondant à la recherche est affichée

Enchaînement nominal :
1. L'utilisateur accède à la page d'accueil.
2. Le système affiche la liste des restaurants disponibles avec
   leur nom, type de cuisine, ville, note et éventuelles promotions.
3. L'utilisateur saisit un mot-clé dans la barre de recherche.
4. Le système filtre et affiche en temps réel les restaurants
   dont le nom ou la cuisine correspond au terme saisi.

Enchaînements d'exception :
E1 : Aucun restaurant ne correspond à la recherche
     → Le système affiche un message indiquant qu'aucun résultat n'a été trouvé.

UC6 – RÉSERVER UNE TABLE
================================================================

Élément              Description
─────────────────────────────────────────────────────────────
Cas d'utilisation    Réserver une table
Acteur principal     Utilisateur connecté
Pré-conditions       L'utilisateur est connecté et consulte la fiche d'un restaurant
Post-conditions      La réservation est enregistrée avec le statut "En attente"

Enchaînement nominal :
1. L'utilisateur clique sur le bouton "Réserver" sur la fiche du restaurant.
2. Le système affiche le formulaire de réservation.
3. L'utilisateur choisit la date, l'heure et le nombre de couverts.
4. L'utilisateur peut ajouter une demande spéciale (allergies, préférences, etc.).
5. L'utilisateur valide la réservation.
6. Le système enregistre la réservation.
7. Le système affiche un message de confirmation.
8. L'utilisateur est redirigé vers son espace compte où la réservation apparaît.

Enchaînements d'exception :
E1 : Date ou heure non disponible → message d'erreur.
E2 : Nombre de couverts invalide (0 ou trop élevé) → message d'erreur.

Acteur principal     Utilisateur connecté
Pré-conditions       L'utilisateur est connecté et consulte la fiche d'un restaurant
Post-conditions      L'avis est enregistré en attente de validation

Enchaînement nominal :
1. L'utilisateur accède à l'onglet "Avis" du restaurant.
2. L'utilisateur clique sur le bouton "Donner mon avis".

Enchaînements d'exception :
E1 : Commentaire vide → message d'erreur, retour à la saisie.
E2 : Note non sélectionnée → message d'erreur.

────────────────────────────────────────
UC9d – Voir mes coupons
────────────────────────────────────────
Élément              Description
─────────────────────────────────────────────────────────────
Cas d'utilisation    Voir mes coupons
Acteur principal     Utilisateur connecté
Pré-conditions       L'utilisateur est connecté et possède suffisamment de points
Post-conditions      Les coupons disponibles sont affichés selon le solde de points

Enchaînement nominal :
1. Le système affiche les coupons auxquels l'utilisateur a droit
   en fonction de son solde de points :
   - 100 pts → Réduction de 10 DT sur la prochaine addition
   - 250 pts → Une entrée offerte lors de la prochaine visite
   - 500 pts → Un plat principal offert au choix
   - 1000 pts → Menu dégustation complet pour 2 personnes
2. Chaque coupon affiche son code, sa valeur et sa description.

Enchaînements d'exception :
E1 : L'utilisateur n'a pas encore assez de points pour débloquer un coupon
     → Le système affiche un message l'encourageant à accumuler des points.

────────────────────────────────────────
UC9e – Insérer un code fidélité
────────────────────────────────────────
Élément              Description
─────────────────────────────────────────────────────────────
Cas d'utilisation    Insérer un code fidélité
Acteur principal     Utilisateur connecté
Pré-conditions       L'utilisateur est connecté et accède à la page Fidélité
Post-conditions      Les points correspondants au code sont ajoutés au solde de l'utilisateur

Enchaînement nominal :
1. L'utilisateur saisit un code promotionnel dans le champ prévu à cet effet.
2. L'utilisateur clique sur "Valider le code".
3. Le système vérifie la validité du code.
4. Si le code est valide, le système ajoute les points correspondants,
   met à jour le niveau si nécessaire et affiche un message de succès.

Enchaînements d'exception :
E1 : Le code est invalide
     → Le système affiche "Code invalide. Veuillez réessayer."
E2 : Le code a déjà été utilisé par cet utilisateur
     → Le système affiche "Vous avez déjà utilisé ce code !"
